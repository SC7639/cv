Event Guide (2020 - 2021):

Viewable at: https://www.skiddle.com/whats-on/all

The Skiddle event guide is a page that queries an API for events and displays the most relevant events to a user based on their chosen filters. The page is served from a custom ssr server backend, built using nodejs and express (as nextjs was mostly for static site generation at the time). It uses react with mui ui library (formerly material ui) and redux for the frontend, which queries an API backed by elastic search. The filters query other apis and some were built using lumen (a stripped-down version of laravel), I built the API that serves the location filter from scratch and is backed again by elastic search for speedy results, it uses a repository pattern that allows any data service to be plugged in a provide data. By default, it can use elastic search or use MySQL as a backup.

Throughout the project, I helped my teammate to learn to react via prs and pair programming sessions, taking advantage of the new at the time vscode live share as the team had moved remote due to covid.

I built a custom SSR server as good SEO was essential for the project. We looked at frameworks like Nextjs but at the time it wasn't useful for dynamic data pages and focused on static site generation. I built a node server that uses express and a worker pool (piscina), written in typescript using ts-node and babel. This is bult, in development and in production using docker/docker compose using a multi-stage Dockerfile. It is deployed using AWS codepipelines that act as ci/cd and the docker image is built and then stored on ECR (elastic contianer registry) and then deployed onto an ec2 instance. The page is cached in production for 1 hour using Cloudfront.

We AB tested using google optimize and we got user feedback using a company called UserTesting during the stages of development/beta testing. With that feedback, we iterated on the design and functionality.

Since this went live, we have rebuilt the project using Nextjs and the newer Skiddle design language, which can be accessed via https://www.skiddle.com/whats-on-beta/all/

The Skiddle ticket box is a react app that uses data attributes to get the parameters required to query the event ticket API. It displays a list of tickets retrieved from the API. It also supports additional views like a calendar for grouped events/sessions. It supports; passwords, discount codes, dependant tickets and tabbed view.

The project was written before I started and took over managing it, currently I am the only person that works on it as the code is old and complex. It is written in the old class based react, I've added functional components with hooks where possible when updating/bug fixing on the project.

The way the project is deployed is slightly different than the other projects as promoters embed the ticket box on their websites. It's built using AWS CodePipeline, then the main files are renamed to latest.js/latest.css and then the files are copied to an S3 bucket that is served via Cloudfront
